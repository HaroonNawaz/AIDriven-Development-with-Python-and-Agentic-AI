## How to Use Skill Maker

### Quick Start (3 Steps)

**Step 1: Describe Your Skill**
```
"I want to create a skill that [describe what you want]"

Examples:
- "validates JSON files and provides error reporting"
- "analyzes code performance bottlenecks"
- "generates documentation from code comments"
```

**Step 2: Answer Clarifying Questions**

Skill Maker asks questions in priority order:

**Priority 1 - Core Functionality** (Must Answer):
- What is the primary function?
- What problem does it solve?
- What are inputs and outputs?

**Priority 2 - Scope & Use Cases** (Must Answer):
- What are typical use scenarios?
- Is it focused on one capability?
- Are there constraints?

**Priority 3 - Tool Requirements** (Must Answer):
- Which Claude Code tools needed?
- Any security considerations?
- Tool restrictions needed?

**Priority 4 - Documentation** (Should Answer):
- Can you provide examples?
- What about edge cases?
- Common issues/solutions?

**Priority 5 - Integration** (Optional):
- Existing infrastructure?
- Related materials?
- Dependencies?

**Step 3: Use Your Generated Skill**
```
New skill location: .claude/skills/[skill-name]/
Contains: SKILL.md + supporting files
Immediately available in Claude Code
```
